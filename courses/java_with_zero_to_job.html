About the course


The main goal of the course is to provide students with the theoretical knowledge and practical skills of Java development, required to work as a Java Developer.
The course is focused on practical skills. Each task solution is checked personally by the mentor.


The course represents itself a strong modular system, each module includes:
  • Practical tasks to strengthen the knowledge
  • Theory exams on studied topics. The knowledge is checked verbally and personally by the mentor
  • Solving test tasks from real interviews
  • Video instructions
  • Reference Guides
Moreover, at the end stage you will develop a final complex project containing all the libraries and frameworks mentioned in the course.

Requirements for students

To enter the course it is necessary to have a basic understanding of programming in general and knowledge of its basic concepts (such as «variable», «function», «conditional statements», «cycles» and «arrays») in particular.

The training process

Training is executed only individually. The entire training is totally adapted to a software development process in a real company. Initially, a student gets access to Bug Tracker. This is the main useful tool at development of projects. And this feature is described in details in the following video. Within the Bug Tracker a student creates a separate folder for all training activities on tasks completion.

Structural plan of the training:
  1. View a video-lesson. Video-lessons are available only to students. All video-lessons are stored on Yandex.Disk.
    Each lesson is represented by one or two videos (duration: 30-60 minutes)
  2. Remember studied information. Draw a brief summary
  3. Perform tasks:
    1. Read the offered task. All tasks are on Bug Tracker.
    2. Think out logic of work. Schematically depict the process of executing the program.
    3. Implement the code. Create a JUnit test.
    4. Check correctness of work in the console by commands mvn clean test
    5. Upload your code to repositories on GitHub
    6. Create a task on Bug Tracker. Specify the corresponding link in it to GitHub
    7. Perform the operation-comment with instructions of the responsible mentor.
    8. Within 1-2 days the task will be checked. The check notification will be sent to your email.
    9. If there are no further fixing instructions in the mentor's reply, the task can be closed and the student can solve the next task.
    10. If the student receives some mentor's fixing instructions, all errors must be corrected, after that the student starts again from the item (h).
  4. Successfully passing each module a student comes to the following section - test questions and a test task. This section simulates a real job interview.
    How to cope with it:
    1. Prepare answers to questions. The section contains a list of questions.
    2. Remember corresponding answers.
    3. Create a task in the tracker (specify date and time of a verbal exam).
    4. On the mentioned day the mentor calls to the student and verbally examined his/her knowledge.
    5. If the student answers at least 80% of the questions, then the interview is considered successful, otherwise the student prepares to repeated exams, until successful outcome.
    6. Successfully passing theory exams, the student performs the test task according to the scheme of Item 3.
  5. During the training you will have some questions. All questions must be entered in the tracker with the name of the responsible person. During the corresponding day you will receive a reply with the time of consulting on Skype.

To summarize: the major part of the training course represents itself a text mode of communication. The mentor calls a student to examine his/her theory knowledge verbally. Also all students can communicate using common HipChat, in which separate chat rooms are divided topically by corresponding training modules. In the chat you can discuss topics related to programming, the course in whole, demonstrate your solutions, ask for solution advices and share useful information.

Upon the training course completion you will have a complex project with all the technologies listed in the course description. Such a project is a significant asset to your resume. Also, when applying for a job, you can mention my contact info in your resume.

Course duration

The course duration is not connected with its cost. Any way, each student must master the entire course content. The length of the training depends on the initial level of knowledge of a particular student and corresponding educational abilities. For example, the stage «Start» - (2-3 lessons per week) can be completed for 2 months. But if you need more than 2 months, you pay nothing additionally, but continue to perfect yourself.
